Project Overview

Price Compare is a website that helps users to search for item and compare the price of the item across three stores, viz. Amazon, eBay and Walmart.

 

Design Features:

The standards and practices followed while working on this project:

·         Fully Responsive - Can be viewed across mobile, tablet, and desktops.

·         Separate files for JavaScript and Style Sheets.

·         Proper error handling.

 

Functional Features:

The main features are:

·         Price Compare helps users to search for item and compare the price of the item across three stores, viz. Amazon, eBay and Walmart.

·         User can login using login credentials. After login, the user is redirected to the profile page.

·         Comparisons and products can be saved to the user profile.

·         When looking up for an item across the stores, the previous saved items on that store are displayed.

·         By clicking on the comparisons or the product, the results of the comparisons or product can be obtained.

Profile

The profile of the user is made of three components.

 

The first part lists the Profile of the user:

·         Username

·         First Name

·         Last Name

·         Email

·         Password

 

The second part lists the Saved Comparisons. The comparisons the user saves are displayed here. When one particular comparison is clicked upon, the results for that comparison are displayed.

The third part lists the Saved Products at particular stores. Again, when a particular product at a store is clicked, the results for the particular product at that store are displayed.

 

How does the page work?

·         On page load, the username is obtained from the Session.                

·         The [ProjectUserTable] is queried with this username and the data corresponding to the username is obtained from the table and is displayed in Profile.

·         The [ProjectComparisonTable] is queried with the username and the 'Saved Comparisons' for the particular username are fetched and displayed in Saved Comparisons.

The [ProjectProductSearch] is queried with the username and the 'Saved Products' and 'Stores' for the particular username are fetched and displayed in Saved Products. 

Search for an item

This page can be navigated to by clicking on the 'Search' button in the Navigation Bar.

 

The page carries the logo on the top left of the page and the social icons on the top right.

 

The search operation is as follows:

·         The user can enter the product that he/she wishes to search and click on the 'Search' button.

·         On click, of the search button, the product is fetched from the Amazon database using the API call and is displayed.

·         The name of the products along with an image are displayed.

·         The user can select 'Compare prices' for the exactly matching product.

 

How does the page work?

·         The user enters the product he/she wants to search for.

·         On click of 'Search', the AngularJS function is called

·         The API page is called, which in turns calls the Amazon API

·         The Amazon API returns XML data

·         This data is passed back to the AngularJS function

·         The XML data is converted to JSON

·         This JSON data is iterated by AngularJS

Compare Prices

When the user selects the product and clicks on 'Compare price', the user is redirected to the Price Comparison page.

 

In this page, on the left, the image of the product the price of which is being compared is displayed.

 

On the right, title of the product is displayed. Right below the product title, a table containing the following fields is displayed:

·         Name of the product.

·         The price of the product.

·         The logo of the store offering the product at the displayed price.

·         A link to fetch detailed info from the store.

The user can save the product being compared to his/her profile by clicking on the 'Save Comparison' button.

 

How does the page work?

·         The name of the product the user wants to do price comparison is passed to the page as a GET parameter.

·         This name is passed to the AngularJS function. The AngularJS function calls the API page.

·         The API page handles the Amazon, eBay and Walmart API calls

·         The data is passed back to the AngularJS function.

·         The data from the API is handled using AngularJS and the required values are iterated.

·         When the user clicks on 'Detailed Info', the name of the product and the store from which the detailed info is requested is passed.

·         On click of 'Save Comparison', the name of the product is inserted into the [ProjectComparisonTable] table.

Detailed Info - Amazon / eBay / Walmart

When the user selects the product at a particular store and clicks on 'Detailed Info', the user is redirected to the Detailed Info of the product at that particular store.

 

In this page, detailed info of the product is fetched from the requested store and displayed to the user. Some of the field are:

·         Name

·         Features / Description

·         Image

·         Category

·         Price

·         View on the item on respective store

 

The product can be saved to the user profile by clicking on 'Save Product'. Also, previously saved products on that store are displayed in the page.

 

How does the page work?

·         The name of the product the user requests a detailed info is passed to the respective page.

·         This name is passed to the AngularJS function. The AngularJS function calls the API page.

·         The API page handles the respective API calls

·         The data is passed back to the AngularJS function.

·         The data from the API is handled using AngularJS and the required values are iterated.

·         On click of 'Save Comparison', the name of the product and the store is inserted into the [ProjectComparisonTable] table.

Sample Use Case

·         Login to the application

·         Enter the item you want to search for - let us consider, "I pad"

·         Now select the item you want to compare the prices for - let us consider, "Apple iPad Mini MF432LL/A (16GB, Wi-Fi, Space Gray)" and click 'Compare Prices'

·         Next add the Apple iPad Mini MF432LL/A (16GB, Wi-Fi, Space Gray ) to the Saved Comparisons by clicking on 'Save Comparison'.

·         Let us view this item on Amazon, for a detailed info. Click on the 'Detailed Info' link.

·         The detailed info of the product from Amazon is displayed.

·         Save this product on Amazon store by clicking on 'Save Product'.

·         Next, go to the user profile, by selecting 'Profile'.

·         The user profile is displayed, along with the Saved Comparisons and Saved Products
